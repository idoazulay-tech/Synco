המטרה העסקית-לוגית של השכבה השנייה
השכבה מקבלת input text ומחזירה "Intent Analysis" שמספיק טוב כדי ששכבות 3 עד 7 יוכלו לעבוד מלא:
- Decision Engine יוכל להחליט execute/ask/reflect/stop
- Task & Time Engine יקבל entities ותלויות
- Learning Engine יקבל החלטות + confidence + context signals
- Feedback Layer יקבל הסבר למשתמש מה הובן ומה חסר

חובה לבנות Intent & Context Engine כמערכת שמורכבת מ-10 רכיבים פנימיים
המודול הראשי: IntentContextEngine
ובתוכו pipeline קבוע:

הערה חשובה:
אם אין ישות מסוימת, עדיין להחזיר אותה עם ערכי ברירת מחדל ו-confidence 0, כדי שכל השכבות הבאות יוכלו לסמוך על מבנה קבוע.

Context Memory (חובה)
לבנות ContextManager שמחזיק "Session State" בזיכרון (in-memory) בלבד בשלב הזה.
הוא צריך לשמור:
- lastIntent
- lastEntities
- lastMissingInfo
- lastUserChoice אם היה
- lastTurnId
- rollingCognitiveLoad (ממוצע קצר)

חוקים:
1) אם המשתמש כותב "כן" או "לא" או "בדיוק" או "ככה" זה כנראה follow-up.
2) אם inputType הוא correction, צריך לקשר לתוצאה הקודמת.
3) ContextManager צריך להציע ל-IntentEngine האם זו השלמה של מידע חסר.

Input Type Classification (חובה חוקים מפורטים)
classifyInputType צריך לעבוד לפי שילוב:
- keywords
- punctuation
- length
- presence of verbs
- correction cues

כללים בסיסיים:
command:
  ביטויים כמו: "תכניס", "תקבע", "שים", "תוסיף", "תזכיר", "תסדר"
thought:
  "אני צריך", "נראה לי", "אולי", "חושב ש"
question:
  סימן שאלה או מילות שאלה: "למה", "איך", "מתי", "איפה", "מה"
correction:
  "לא", "לא ככה", "עזוב", "בעצם", "תיקון", "שכח מזה"
emotional_dump:
  הרבה רגש, מילים כמו "לחוץ", "נמאס", "לא מסוגל", "אני קורס", וגם טקסט ארוך ללא בקשת פעולה ברורה

Intent Detection (חובה)
detectIntents צריך לזהות primaryIntent + secondaryIntents.
דוגמאות:
- "תכניס לי פגישה" => create_event
- "תזכיר לי" => create_task או remind (אבל בשלב הזה לתרגם ל-create_task עם entity must/urgency)
- "תסדר לי את היום" => reschedule
- "לבטל" => cancel
- "מה יש לי היום" => inquire

אם יש ambiguity, primaryIntent יכול להיות unknown אבל אז confidenceScore יורד וה-missingInfo יכלול "clarification_needed".

Entity Extraction (חובה, בלי ספריות חיצוניות בשלב MVP)
extractEntities צריך להיות Rule Based עם patterns ו-keywords.
הוא צריך לתמוך לפחות ב:
Time:
  "ב-2", "בשעה 14:00", "שתיים וחצי", "בעוד שעה", "עוד 20 דקות"
Date:
  "היום", "מחר", "מחרתיים", "ביום ראשון", "ביום חמישי", "השבוע", "מחר בבוקר"
Duration:
  "20 דקות", "חצי שעה", "שעה", "שעתיים"
People:
  אחרי "עם", או זיהוי שם פרטי בודד (הכי פשוט: מילה שמתחילה באות גדולה באנגלית, ובעברית לזהות לאחר "עם")
Location:
  אחרי "ב", "בבית", "בעבודה", "ברמת גן", "במשרד"
Task Name:
  אם יש פועל ועוד אובייקט: "לקנות מים", "לשטוף כלים", "להכין אוכל"
Urgency:
  מילים: "דחוף", "עכשיו", "מייד", "חייב"
Must:
  מילים: "חייב", "אין מצב שלא", "must", "נעול"

ייצוג זמן ותגיות
- להחזיר raw תמיד
- iso רק אם אפשר לחשב בביטחון. אם לא, להשאיר iso ריק ו-confidence נמוך.
- אין צורך Timezone חיצוני. אבל כן להגדיר מקום בעתיד.

Commitment Level (חובה)
detectCommitmentLevel לפי ניסוח:
high:
  "תעשה", "תכניס", "אני חייב", "עכשיו", "תסדר"
medium:
  "כדאי", "אפשר", "בוא", "אני רוצה"
low:
  "אולי", "נראה לי", "חושב ש", "מתישהו"

Cognitive Load (חובה)
detectCognitiveLoad מחזיר low/medium/high לפי:
- אורך טקסט
- כמות פסיקים
- חזרות
- מעבר נושאים מהיר
- מילים כמו "דחוף דחוף דחוף"
דוגמה:
אם יש חזרות "דחוף" או טקסט מבולגן => high

Missing Critical Info (חובה לפי Intent)
detectMissingInfo צריך כללים לכל intent:
create_event:
  חייב: date + time + people או title
create_task:
  חייב: taskName
  מומלץ: duration או deadline
reschedule:
  חייב: scope (היום/מחר/המשימות האלה)
cancel:
  חייב: target (מה מבטלים)
inquire:
  לא חייב כלום, אבל יתכן scope

Confidence Score (חובה)
computeConfidenceScore מחזיר 0 עד 1 לפי:
- כמה מזהים בוודאות inputType + primaryIntent
- כמה entities נמצאו
- כמה missingInfo נשאר
- האם יש conflict בין signals
להגדיר scoring weights ב-/rules/scoring.ts

Explainability (חובה פנימי)
explainability.ts צריך לייצר notes פנימיים:
למשל:
- "Detected command due to keyword 'תכניס'"
- "Primary intent create_event from phrase 'פגישה'"
- "Missing date"
זה לא בהכרח מוצג למשתמש, אבל חייב להיות קיים לאבחון ולשכבת Feedback בעתיד.
