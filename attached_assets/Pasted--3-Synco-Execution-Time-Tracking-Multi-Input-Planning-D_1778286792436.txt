חלק 2: שירות חישוב זמן ביצוע

צור קובץ חדש:
server/services/executionTrackingService.ts

הוא יכיל פונקציות:

calculateActualDurationMinutes(actualStartTime, actualEndTime)
calculateDurationDeltaMinutes(plannedDurationMinutes, actualDurationMinutes)
buildExecutionCompletedMetadata(task, actualStartTime, actualEndTime)

כללים:
1. אם actualEndTime לפני actualStartTime, אל תחזיר duration שלילי.
2. אם plannedDurationMinutes חסר, שמור actualDurationMinutes אבל durationDeltaMinutes יהיה null.
3. אם actualStartTime חסר, אל תקרוס.
4. אם יש כמה task_started לאותה משימה, בשלב הזה השתמש באחרון לפי occurredAt.
5. task_execution_completed כפול לא עוצר את המערכת. רק תיעוד. dedup יבוא בעתיד.

חלק 4: taskStore

עדכן בזהירות:
src/store/taskStore.ts

הוסף אם אין:
- startTaskExecution(taskId)
- getTaskExecutionStart(taskId)
- clearTaskExecutionStart(taskId)

ה-start בפועל יכול להישמר ב-localStorage/store כמצב זמני:
executionStarts: Record<taskId, actualStartTimeIso>

כאשר המשתמש מתחיל משימה:
1. שמור actualStartTimeIso.
2. שלח task_started.
3. אל תשנה status ל-completed.
4. אל תשבור UI קיים.

כאשר המשתמש מסמן משימה כהושלמה:
1. המשך ליצור task_completed כמו היום.
2. בדוק אם יש actualStartTime שמור.
3. אם יש, חשב actualDurationMinutes.
4. אם אין, נסה להשתמש ב-startTime המתוכנן כ fallback רק אם זה הגיוני.
5. שלח task_execution_completed עם metadata.
6. נקה actualStartTime מה-store.
7. אם לוגינג נכשל, completeTask עדיין מצליח.

חשוב:
PlanningDraftTask לא מקבל startTaskExecution.
רק UserTask אמיתי.

חלק 6: הרחבת Daily Learning Summary

עדכן:
server/services/learningSummaryService.ts

הוסף ל-response:

executionStats:
{
  "trackedCompletedCount": 2,
  "averagePlannedDurationMinutes": 45,
  "averageActualDurationMinutes": 61,
  "averageDeltaMinutes": 16,
  "overranCount": 2,
  "underranCount": 0
}

כללים:
1. חשב רק מאירועי task_execution_completed שיש להם metadata.actualDurationMinutes.
2. plannedDurationMinutes מגיע מתוך metadata.
3. durationDeltaMinutes מגיע מתוך metadata או מחושב.
4. overranCount: actualDurationMinutes > plannedDurationMinutes.
5. underrunCount: actualDurationMinutes < plannedDurationMinutes.
6. אם אין נתונים, החזר:
executionStats:
{
  "trackedCompletedCount": 0,
  "averagePlannedDurationMinutes": null,
  "averageActualDurationMinutes": null,
  "averageDeltaMinutes": null,
  "overranCount": 0,
  "underranCount": 0
}

אל תשבור את response הקיים של daily-summary.
