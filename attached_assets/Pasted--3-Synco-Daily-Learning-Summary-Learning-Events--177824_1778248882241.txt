השירות צריך לייצא פונקציה:

buildDailyLearningSummary(userId: string, dateIso: string)

הפונקציה:
1. שולפת LearningEvents מה-DB לפי userId ו-dateIso.
2. אם dateIso חסר בחלק מהאירועים, אפשר להשתמש ב-occurredAt לפי תחילת וסוף היום.
3. מחשבת counts לפי eventType.
4. מחשבת completionHours לפי task_completed.
5. מחשבת mostMovedTasks לפי task_rescheduled.
6. מחזירה notes פשוטים בעברית.
7. לא משתמשת ב-AI.
8. לא כותבת שום דבר ל-DB בשלב הזה.

Route:
בתוך server/routes/learning.ts הוסף:

GET /daily-summary

או אם ה-router מחובר תחת /api/learning:
הנתיב הסופי צריך להיות:

GET /api/learning/daily-summary?userId=default-user&date=2026-05-05

Validation:
1. userId חובה.
2. date חובה בפורמט YYYY-MM-DD.
3. אם חסר userId או date, החזר 400.
4. אם אין אירועים, החזר ok:true עם summary ריק ולא שגיאה.
5. אם יש שגיאת DB, החזר 500 עם הודעה פשוטה.

Response כאשר אין אירועים:
{
  "ok": true,
  "userId": "default-user",
  "date": "2026-05-05",
  "summary": {
    "createdCount": 0,
    "completedCount": 0,
    "rescheduledCount": 0,
    "deletedCount": 0,
    "scheduleAppliedCount": 0,
    "totalEvents": 0,
    "completionHours": [],
    "mostMovedTasks": [],
    "taskIdsTouched": [],
    "notes": ["לא נמצאו אירועי למידה ליום הזה"]
  }
}
